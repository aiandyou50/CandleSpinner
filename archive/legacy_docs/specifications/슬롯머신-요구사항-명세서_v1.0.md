## 1. 전체 게임 규칙 및 구조

### 1.1 슬롯머신 규칙 (Slot Machine Rules)

#### 릴(Reel) 구성
- **3-Reel 슬롯머신**
- 각 릴은 독립적으로 작동
- 수직 정렬, 가로 배치

#### 심볼(Symbols) 정의

이미지 파일 대신 **이모지(Emoji)**를 사용하여 로딩 속도 최적화 및 크로스 플랫폼 호환성 확보

| 심볼 | 명칭 | 희귀도 | 배당률 | 개별 릴 확률 |
|------|------|--------|--------|------------|
| ⭐ | 별 | Common | x0.5 | 35% |
| 🪐 | 행성 | Common | x1 | 25% |
| ☄️ | 혜성 | Uncommon | x2 | 15% |
| 🚀 | 로켓 | Uncommon | x3 | 10% |
| 👽 | 외계인 | Rare | x5 | 7% |
| 💎 | 다이아몬드 | Rare | x10 | 5% |
| 👑 | 왕관 | Legend | x20 | 3% |

**총합**: 100%

#### 당첨금 계산 방식 (독창적 규칙)

**핵심**: 당첨된 각 릴의 상금을 개별적으로 계산하여 모두 합산

```
당첨금 = Σ (베팅액 × 해당 릴의 심볼 배당률)
```

**예시 1**: 베팅액 100 CSPIN
- 릴1: 로켓 (x3) → 300 CSPIN
- 릴2: 로켓 (x3) → 300 CSPIN
- 릴3: 별 (x0.5) → 50 CSPIN
- **총 당첨금**: 300 + 300 + 50 = **650 CSPIN**

**예시 2**: 베팅액 50 CSPIN
- 릴1: 다이아몬드 (x10) → 500 CSPIN
- 릴2: 행성 (x1) → 50 CSPIN
- 릴3: 혜성 (x2) → 100 CSPIN
- **총 당첨금**: 500 + 50 + 100 = **650 CSPIN**

#### 잭팟(Jackpot) 규칙

**발동 조건**: 3개 릴에 모두 동일한 심볼이 나올 경우

**잭팟 상금 계산**:
```
잭팟 상금 = (3개 릴의 개별 당첨금 총합) × 100
```

**예시**: 베팅액 100 CSPIN, 3개 릴 모두 '로켓(x3)' 잭팟
- 릴1: 100 × 3 = 300 CSPIN
- 릴2: 100 × 3 = 300 CSPIN
- 릴3: 100 × 3 = 300 CSPIN
- 개별 합산: 900 CSPIN
- **잭팟 상금**: 900 × 100 = **90,000 CSPIN**

**잭팟 시각 효과**:
- `public/jackpot_video.mp4` 파일을 전체 화면으로 재생
- 자동 재생, 소리 포함, 약 5초 길이
- 비디오 재생 중 모든 UI 비활성화
- 비디오 종료 후 자동으로 결과 화면 전환

#### 공정성 (Provably Fair)

**알고리즘**: 서버 시드 + 클라이언트 시드 기반 해시 검증

1. **서버 시드**: 게임 시작 전 서버에서 생성 (SHA-256 해시로 사전 공개)
2. **클라이언트 시드**: 사용자가 직접 입력하거나 자동 생성
3. **논스(Nonce)**: 각 게임마다 증가하는 카운터
4. **결과 생성**: HMAC-SHA256(서버시드, 클라이언트시드:논스)
5. **검증 가능**: 게임 종료 후 서버 시드 공개하여 사용자가 직접 검증

### 1.2 미니게임 (더블업, Double-Up)

#### 발동 조건
- 슬롯머신 스핀에서 **1회라도 당첨금이 발생하면 항상 등장**
- 베팅액과 무관하게 당첨금 > 0이면 트리거

#### 게임 방식
- **선택지**: 빨간색(❤️) 버튼 또는 파란색(💙) 버튼
- **확률**: 각각 정확히 50%
- **결과**: 서버에서 난수 생성하여 결정

#### 보상 구조
- **성공**: 현재 당첨금 × 2
- **실패**: 현재 당첨금 전액 손실 (0 CSPIN)

#### 플레이 제한
- **슬롯머신 스핀 1회당 단 한 번의 더블업 기회만 주어짐**
- 연속 더블업 불가 (무한 증식 방지)
- 더블업 성공 후 다시 슬롯 스핀부터 시작

**예시 시나리오**:
1. 슬롯 스핀 → 300 CSPIN 당첨
2. 더블업 팝업 등장 → 파란색 선택 → 성공
3. 당첨금 600 CSPIN으로 증가
4. 더블업 종료 → 다음 스핀 가능

### 1.3 토큰 경제 모델 (Tokenomics)

#### RTP (Return to Player): 95%

**의미**: 전체 사용자 베팅액의 95%가 장기적으로 상금 풀(Pool)로 사용자에게 반환

**설계 근거**:
- 심볼 확률과 배당률을 조정하여 이론상 RTP 95% 달성
- 10,000회 이상 시뮬레이션으로 검증 필요

**RTP 계산 공식**:
```
RTP = (총 지급 상금) / (총 베팅액) × 100%
```

#### 하우스 엣지 (House Edge): 5%

**의미**: 프로젝트의 지속적인 운영, 개발, 마케팅을 위한 수익 모델

**활용 계획**:
- 서버 인프라 유지 비용
- 개발팀 운영비
- 마케팅 및 사용자 보상 프로그램
- TON 블록체인 트랜잭션 수수료 보전

### 2.2 슬롯머신 인터페이스

#### 릴 디자인

**기본 스펙**:
- 3개의 수직 릴 배치 (가로 정렬)
- 각 릴 크기: 120x240px (데스크톱 기준)
- 심볼 표시 영역: 각 릴당 3개의 이모지 심볼 동시 표시
- 구분선: 릴 사이에 명확한 경계선 (2px, #4A4A4A)

**구조**:
```
┌───────┬───────┬───────┐
│  ⭐   │  🚀   │  💎   │  ← 상단 심볼
├───────┼───────┼───────┤
│  🪐   │  👽   │  ☄️   │  ← 중앙 (당첨 라인)
├───────┼───────┼───────┤
│  🚀   │  💎   │  👑   │  ← 하단 심볼
└───────┴───────┴───────┘
```

#### 심볼 시각화 규칙

**희귀도별 시각적 구분**:

1. **Common** (⭐, 🪐):
   - 일반 테두리
   - 흰색 배경 (#FFFFFF)
   - 그림자 없음

2. **Uncommon** (☄️, 🚀):
   - 파란색 하이라이트 (#4A90E2)
   - 약간의 그림자 효과 (box-shadow: 0 2px 8px rgba(74,144,226,0.3))
   - 미세한 발광 효과

3. **Rare** (👽, 💎):
   - 보라색 하이라이트 (#9B59B6)
   - 부드러운 발광 효과 (box-shadow: 0 0 15px rgba(155,89,182,0.5))
   - 애니메이션 (미세한 펄스)

4. **Legend** (👑):
   - 금색 하이라이트 (#F1C40F)
   - 강렬한 발광 효과 (box-shadow: 0 0 25px rgba(241,196,15,0.8))
   - 애니메이션 (회전 + 펄스)

**당첨 라인**:
- 중앙 수평선에 노란색 (#FFD700) 강조선 표시
- 두께: 3px
- 애니메이션: 부드러운 펄스 효과

**스핀 중 애니메이션**:
- 부드러운 스크롤 효과
- 감속 곡선(ease-out) 적용
- 릴별로 0.2초 간격으로 정지

**결과 애니메이션**:
- 당첨된 심볼 주변에 펄스 효과 (0.5초)
- Scale 변화: 1.0 → 1.1 → 1.0
- 당첨 라인 깜빡임 효과

#### 잭팟 애니메이션

**트리거**: 3개 릴 동일 심볼

**효과 순서**:
1. 화면 전체 진동 효과 (0.5초)
2. 불꽃 애니메이션 (파티클 효과)
3. `public/jackpot_video.mp4` 전체 화면 재생
4. 비디오 재생 중 모든 UI 비활성화
5. 비디오 종료 후 결과 화면 전환 (상금 표시)

**비디오 스펙**:
- 포맷: MP4 (H.264)
- 해상도: 1920x1080 (16:9)
- 길이: 약 5초
- 자동 재생, 소리 포함
- 로딩 최적화: 사전 캐싱

### 2.3 상호작용 요소

#### 2.3.1 베팅 컨트롤

**베팅 슬라이더**:
- **범위**: 10 ~ 1,000 CSPIN
- **단계**: 10 CSPIN 단위 (증분)

  - 원형 핸들 (드래그 가능)

**퀵 베팅 버튼**:
- **MAX 버튼**: 최대 보유 금액 베팅 (한 번 클릭)
- **단축 버튼**: 100, 500, 1000 CSPIN 버튼 (3개)
- **버튼 디자인**:
  - 평면 스타일 (Flat Design)
  - 호버 시 상승 효과 (translateY: -2px)
  - 클릭 시 리플 효과

**상태 표시**:
- 잔액 부족: 버튼 비활성화 + 붉은색 테두리
- 최소/최대 제한: 슬라이더 끝에 표시

#### 2.3.2 스핀 버튼

**디자인**:
- **크기**: 80x80px, 원형
- **색상**:
  - 활성화: #4CAF50 (녹색)
  - 비활성화: #9E9E9E (회색)
- **아이콘**: 중앙에 화살표 아이콘 (회전)
- **애니메이션**:
  - 호버: 확대 효과 (scale: 1.05)
  - 클릭: 펄스 효과 + 0.1초 회전
  - 스핀 중: 로딩 인디케이터 (회전하는 원)

**상태 표시**:
- **충분한 잔액이 없을 경우**:
  - 버튼 비활성화
  - 알림 텍스트: "잔액이 부족합니다" (붉은색)
- **게임 진행 중**:
  - 버튼 비활성화
  - 로딩 인디케이터 표시

**키보드 단축키**:
- 스페이스바: 스핀 (활성화 상태일 때)

## 3. 기능 요구사항 정의

### 3.1 슬롯머신 핵심 기능

**FR-S001**: 시스템은 3개의 수직 릴을 표시하며, 각 릴은 명세서에 명시된 7개의 이모지 심볼(⭐, 🪐, ☄️, 🚀, 👽, 💎, 👑)을 표시할 수 있어야 한다.

**FR-S002**: 당첨금은 각 릴의 결과를 개별적으로 계산하여 합산한다: (베팅액 × 해당 심볼의 배당률)을 릴별로 계산 후 모두 더함.

**FR-S003**: 3개 릴에 동일한 심볼이 등장할 경우 잭팟이 자동 발동되며, 상금은 (개별 당첨금 총합) × 100으로 계산되어야 한다.

**FR-S004**: 잭팟 발생 시 `public/jackpot_video.mp4` 파일이 전체 화면으로 재생되어야 한다.

**FR-S005**: 시스템은 Provably Fair 알고리즘을 통해 게임 결과의 공정성을 보장하고, 사용자가 이를 검증할 수 있는 기능을 제공해야 한다.

### 3.2 미니게임(더블업) 기능

**FR-D001**: 슬롯 스핀 결과에서 상금이 발생할 경우, 자동으로 더블업 미니게임이 팝업되어야 한다.

**FR-D002**: 더블업 게임은 빨간색과 파란색 버튼 중 하나를 선택하는 방식으로 구현되어야 하며, 성공 확률은 정확히 50%여야 한다.

**FR-D003**: 성공 시 당첨금의 2배를 획득하고, 실패 시 해당 당첨금을 모두 잃어야 한다.

**FR-D004**: 단일 슬롯 스핀 당 최대 1회만 더블업 게임을 진행할 수 있어야 하며, 연속으로 더블업 게임을 할 수 없어야 한다.

### 3.3 토큰 경제 관련 기능

**FR-T001**: 시스템은 장기적으로 RTP 95%를 유지할 수 있도록 설계되어야 한다.

**FR-T002**: 모든 트랜잭션 기록이 실시간으로 저장되어야 하며, 사용자가 확인할 수 있어야 한다.

**FR-T003**: 매일 자정에 당일의 RTP 통계를 계산하여 관리자 대시보드에 표시해야 한다.

## 부록

### A. 심볼 확률 상세 계산

**목표 RTP**: 95%

**각 심볼의 기대값 계산**:
```
기대값 = 확률 × 배당률

⭐: 0.35 × 0.5 = 0.175
🪐: 0.25 × 1 = 0.25
☄️: 0.15 × 2 = 0.30
🚀: 0.10 × 3 = 0.30
👽: 0.07 × 5 = 0.35
💎: 0.05 × 10 = 0.50
👑: 0.03 × 20 = 0.60

총 기대값 (1개 릴): 2.525
3개 릴 총 기대값: 2.525 × 3 = 7.575

이론상 RTP: 7.575 / 8 = 94.69% ≈ 95%
```

*(참고: 잭팟 확률은 매우 낮아 전체 RTP에 미미한 영향)*
