***REMOVED***.gitignore 학습 가이드

#***REMOVED***1. .gitignore란?
`프로젝트에서 Git이 추적하지 않기를 원하는 파일/폴더를 정의하는 설정 파일`

#***REMOVED***2. 주요 패턴 문법

##***REMOVED***기본 패턴
```gitignore
***REMOVED***특정 파일 제외
node_modules/           ***REMOVED***node_modules 폴더 제외
*.log                   ***REMOVED***모든 .log 파일 제외
.env                    ***REMOVED***.env 파일 제외
dist/                   ***REMOVED***dist 폴더 제외

***REMOVED***특정 경로의 폴더 제외
/node_modules           ***REMOVED***루트의 node_modules만 제외 (하위 폴더의 node_modules는 포함)
contracts/node_modules  ***REMOVED***contracts 폴더 내 node_modules 제외
**/node_modules         ***REMOVED***모든 경로의 node_modules 제외 (권장)

***REMOVED***패턴 매칭
*.js                    ***REMOVED***모든 .js 파일 제외
test/**/*.ts            ***REMOVED***test/ 하위의 모든 .ts 파일 제외
src/**/*.spec.ts        ***REMOVED***src 하위의 모든 .spec.ts 파일 제외

***REMOVED***제외 대상에서 특정 파일 포함 (!)
*.log                   ***REMOVED***모든 .log 파일 제외
!important.log          ***REMOVED***important.log는 포함
```

#***REMOVED***4. 제외가 필요한 파일 종류

| 구분 | 파일 | 이유 |
|------|------|------|
| **의존성** | node_modules/ | 패키지 매니저(npm)가 자동으로 생성. 불필요한 용량 차지 (~100MB+) |
| **환경 설정** | .env, .env.local | 민감한 정보 포함 (API 키, 프라이빗 키 등) |
| **빌드 결과** | dist/, build/ | 소스에서 자동 생성 (재빌드 가능) |
| **로그 파일** | *.log, npm-debug.log | 실행 중 생성되는 임시 파일 |
| **시스템 파일** | .DS_Store, Thumbs.db | OS가 자동 생성 |
| **IDE 설정** | .vscode/, .idea/ | 개발자마다 다른 설정 |
| **비밀 키** | *.key, *.private | 보안 위협 (절대 커밋 금지) |

#***REMOVED***5. Git 명령어

##***REMOVED***.gitignore 적용 확인
```bash
***REMOVED***무시되는 파일 목록 확인
git check-ignore -v <filename>

***REMOVED***모든 무시 패턴 확인
git status --porcelain --ignored

***REMOVED***특정 패턴으로 무시되는 파일만 표시
git ls-files -o -i --exclude-standard
```

##***REMOVED***이미 커밋된 파일 제외하기
```bash
***REMOVED***1. .gitignore에 패턴 추가
echo "node_modules/" >> .gitignore

***REMOVED***2. Git 캐시에서 제거 (파일은 유지)
git rm --cached -r node_modules/

***REMOVED***3. 변경사항 커밋
git commit -m "Remove node_modules from tracking"

***REMOVED***4. 변경사항 푸시
git push origin main
```

#***REMOVED***6. CandleSpinner의 .gitignore 수정 내역

##***REMOVED***변경 사항
```diff
***REMOVED***Before
/node_modules

***REMOVED***After
/node_modules
/contracts/node_modules      ← 명시적으로 추가
contracts/node_modules       ← 이중 안전장치
**/node_modules              ← 모든 경로 대응
```

##***REMOVED***이유
1. **명시성**: contracts/node_modules 명확히 제외
2. **안전성**: 다양한 패턴으로 중복 방지
3. **미래 대비**: 새로운 서브디렉토리 자동 적용

#***REMOVED***8. 베스트 프래틱스

##***REMOVED***✅ DO
- `.gitignore`를 프로젝트 루트에 배치
- 컨벤션 사용 (node_modules/, dist/ 등)
- 환경 변수 파일 제외 (.env)
- 비밀 정보 제외 (*.key, *.private)
- 주석으로 섹션 구분

##***REMOVED***❌ DON'T
- `.gitignore` 자체는 커밋 (프로젝트 설정 공유 필요)
- 개인 IDE 설정은 전역 .gitignore 사용
- 패턴 없이 임의로 파일 추가
- 이미 커밋된 대용량 파일만 제외 (효과 없음)

#***REMOVED***🎯 핵심 정리

1. **목적**: 불필요하고 민감한 파일을 Git 추적 대상에서 제외
2. **문법**: 패턴 매칭 (와일드카드, 경로 포함)
3. **적용**: 루트 .gitignore에 정의
4. **확인**: `git status`, `git check-ignore` 명령어 사용
5. **순서**: .gitignore 작성 → 커밋 → 푸시 (이전 파일은 `git rm --cached` 필요)
